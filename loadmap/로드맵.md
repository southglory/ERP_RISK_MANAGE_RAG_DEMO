# AI 회계·감사 Playground 빌드업 로드맵

> **프로젝트 정체성 (2026-05-07)**: **AI 회계·감사 Playground**. 회계·세무 룰 엔진 + 법령·판례 RAG + 거래 데이터 + 거버넌스 추적을 결합한 통합 감사 시스템 시뮬레이터.
>
> **목적**: 회계·세무 도메인의 복잡한 규칙을 소프트웨어적으로 체계화하고, LLM과 RAG를 활용하여 의사결정의 신뢰도를 높이는 기술 스택 빌드업.
>
> **읽는 법**: 각 작업은 3가지로 쪼개져 있음.
> - **만든 거**: 손에 잡히는 결과물 (눈에 보이는 것)
> - **보여줄 모습**: "이런 입력 → 이런 출력" 데모 시나리오
> - **잘됐는지 확인**: 측정 가능한 성공 기준
>
> **로컬 환경**: RTX 3090, 24GB VRAM. 합산 적재 ≤ 22GB.

## 스택 매핑 — 클라우드 ↔ 로컬 스왑 전략

| 서비스 계층 | 로컬 (지금) | 추상 인터페이스 | 스왑 비용 |
|---|---|---|---|
| LLM Service | Solar-10.7B-Instruct AWQ + vLLM | `LLMProvider.chat()` | `BASE_URL` 1줄 |
| Embedding | BGE-M3 (8K, Dense+Sparse) | `EmbeddingProvider.embed()` | 어댑터 1개 |
| Document Parse | PaddleOCR + Donut + LLM 후처리 | `OCRProvider.parse()` | 어댑터 1개 |
| Vector Store | **pgvector** / **Milvus** | `VectorStore.search()` | 어댑터 1개 |
| Reranker | bge-reranker-v2-m3 | `Reranker.rerank()` | 0 (동일 패밀리) |

**UI 레이어**: **PySide6** (LGPL) + Qt Designer + qt-material 다크 테마 + qasync(LLM 스트리밍). 자체 윈도우 기반 데스크톱 프로그램. `core/` ↔ `ui/pyside/` 분리로 GUI 폐기·교체 자유 보장.

**저장소 구조**:
```
core/         # Provider · 룰엔진 · RAG · LangGraph
ui/pyside/    # PySide6 데스크톱 앱
seed/ docs/   # 지식 베이스 자료
eval/         # 평가 결과
```

## 진행 원칙

- **순서 엄수**: Phase 1 미완성 상태에서 Phase 2 시작 금지.
- **Provider 추상화**: 모든 모델 콜은 인터페이스 뒤로 격리하여 유연한 교체 지원.
- **리니지 우선**: 모든 LLM/RAG/룰 콜에서 결정 근거 및 트레이스 ID 전파.
- **룰 우선, LLM은 보조**: 회계·감사 도메인은 답변의 80%를 룰이 풀고, LLM은 보조적 판단 및 자연어 처리 담당.
- **VRAM 효율화**: 합산 VRAM ≤ 22GB 유지를 위한 모델 양자화 및 최적화.
- **데이터 보안 및 라이선스**: 합성 데이터 및 공개 법령 데이터만 사용. 상용 모델 라이선스 준수.
- **core/ui 분리**: 로직(`core/`)과 인터페이스(`ui/`)를 철저히 분리하여 이식성 확보.
